# Process vs Thread
### Process
1. 정의
    - 실행 중인 프로그램
    - 프로그램이 실행되면 운영체제(OS)가 프로세스를 만들어 메모리에 올린다
1. 특징
    - 각각 독립된 메모리 공간을 가진다
    - 다른 프로세스의 메모리에 직접 접근할 수 없다
    - 하나의 프로세스에는 여러 개의 스레드가 존재할 수 있다 
1. 예시
    - 크롬 실행 -> 크롬 프로세스
    - 메모장 실행 -> 메모장 프로세스
    - 즉, 프로그램을 실행하면 프로세스를 생성한다

### Thread
1. 정의
    - 프로세스 안에서 실제 작업을 수행하는 실행 단위(스레드가 일을 해야 프로세스가 동작한다)
1. 특징
    - 하나의 프로세스 안에 여러 개의 스레드가 존재할 수 있다
    - 같은 프로세스의 스레드들은 메모리를 공유한다
    - 동시에 여러 작업을 수행할 수 있다
1. 예시
    - 크롬(프로세스)
        - Thread 1 : 화면 출력
        - Thread 2 : 인터넷 다운로드
        - Thread 3 : 음악 재생
        - Thread 4 : 사용자 입력 처리
        - 하나의 프로세스 안에서 동시에 실행된다

    ### Process vs Thread
    Process | Thread
    ---|---
    실행중인 프로그램 | 실제 작업을 수행하는 실행 단위
    독립적인 메모리 공간 사용 | 같은 프로세스의 메모리 공유
    생성 비용이 큼    | 생성 비용이 적음
    프로세스끼리는 메모리 공유 불가 | 같은 프로세스 내에서는 메모리 공유 가능
    하나 이상의 스레드를 포함 | 반드시 어떤 프로세스 안에 존재
