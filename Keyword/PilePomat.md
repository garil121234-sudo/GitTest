# 파일 포맷
파일 안에 데이터가 어떤 규칙과 구조로 저장되어 있는지를 정한 형식  

**파일 포맷이 필요한 이유**  
컴퓨터는 파일의 내용을 그냥 바이트(0 ,1)의 나열로 저장한다  
(0, 1)을 사용하여 포맷 규칙을 보고 사진,음악,문서 인지 해석한다  
즉, 데이터를 읽고 해석할지 알려주는 약속이다

**파일 포맷의 구성**
```
┌──────────────────────┐
│ Header (헤더)        │
├──────────────────────┤
│ Data (실제 데이터)    │
├──────────────────────┤
│ Footer (선택사항)     │
└──────────────────────┘
```
1. Header  
파일의 정보를 저장한다  
ex)파일종류, 버전, 크기, 이미지의넓이, 압축 방식
1. Data  
실제 내용이 저장된다  
ex)사진의픽셀, 음악의음원, 문서의글자
1. Footer  
필요한 경우 마지막에 추가 정보가 저장된다
ex)종료표시, 체크섬(오류검사), 인덱스

**확장자와 파일 포맷의 차이**  
ex)photo.jpg 에서 .jpg는 확장자이다  
확장자는 "이 파일은 JPEG 형식일 가능성이 놓다"라고 알려주는 이름표  
실제 파일 포맷은 파일 내부의 구조이다  
*확장자 = 파일 이름 뒤에 붙는 표시*  
*파일 포맷 = 파일 내부의 저장 방식*

### 대표적인 파일 포맷
종류|포맷|설명
---|---|---
이미지|JPEG(.jpg)|사진 저장에 많이 사용
이미지|PNG(.png)|투명 배경 지원
이미지|BMP(.bmp)|압축이 거의 없는 이미지
이미지|GIF(.gif)|간단한 애니메이션 지원
이미지|WebP(.webp)|작은 용량과 좋은 화질
문서|TXT(.txt)	|일반 텍스트
문서|PDF(.pdf)	|문서 레이아웃 유지
문서|DOCX(.docx)	|문서 편집용
음악|MP3(.mp3)	|압축 오디오
음악|WAV(.wav)	|무압축 오디오
동영상|MP4(.mp4)	|가장 많이 사용하는 동영상
압축|ZIP(.zip)	|여러 파일 압축
