# JPG(JPEG)
JPG는 사진과 같은 이미지를 저장하기 위해 가장 많이 사용하는 이미지 파일 형식  
- JPEG : Joint Photographic Experts Group의 약자
- JPG : JPEG 파일의 확장자(.jpg)
    - 원래는 .JPEG를 사용했지만 예전 운영체제에서 확장자를 3글자만 사용할 수 있어서 .JPG가 많이 사용되었다
    - 현재는 .JPG 와 .JPEG는 완전히 같은 형식이다

### 특징
1. 손실 압축  
가장 큰 특징은 손실 압축이다  
이미지 용량을 줄이기 위해 사람이 잘 알아보기 어려운 일부 데이터를 삭제한다  
용량이 작아지지만 원본과 100% 동일하지는 않다
```
원본 이미지
      ↓
압축
      ↓
일부 정보 삭제
      ↓
JPG 파일
```
1. 파일 용량이 작다  
같은 사진이라도 JPG가 훨씬 작은 경우가 많다.  
ex) BMP : 15MB  
PNG : 8MB  
JPG : 2MB  
그래서 인터넷, 스마트폰 사진, 웹사이트 에서 많이 사용된다
1. 사진에 적합하다  
색이 많이 섞인 이미지는 압축 효율이 매우 좋다.  
ex)사람 사진  
풍경 사진  
음식 사진  
게임 스크린샷  
1. 투명 지원 안 함  
JPG는 투명 배경(알파 채널)을 지원하지 않는다
```
PNG
□□□□
□  ○  □
□□□□ 배경 투명 가능

JPG
■■■■
■ ○ ■
■■■■ 배경이 흰색 또는 다른 색으로 저장됨

```
1. 여러 번 저장하면 화질이 떨어진다  
JPG는 저장할 때마다 다시 압축한다  
```
원본
 ↓
저장
 ↓
화질 조금 감소
 ↓
다시 저장
 ↓
또 감소
 ↓
반복
```
그래서 편집을 여러 번 하면 화질이 계속 떨어질 수 있다  

**장점**
- 용량이 매우 작다
- 사진 저장에 적합하다
- 대부분의 프로그램에서 지원한다
- 웹에서 가장 많이 사용되는 이미지 형식 중 하나이다

**단점**
- 손실 압축이라 원본 정보가 일부 사라진다
- 여러 번 저장하면 화질이 계속 감소한다
- 투명 배경을 지원하지 않는다
- 글자나 아이콘처럼 선명한 그래픽에는 적합하지 않다

**사용하는 경우**
- 디지털 카메라 사진
- 스마트폰 사진
- 웹사이트 이미지
- SNS 업로드
- 블로그 사진

**사용하지 않는 경우**
- 로고
- 아이콘
- 투명 이미지
- 그림판 작업
- 자주 수정해야 하는 이미지(이 같은 경우는 PNG가 적합)

**JPG와 PNG 비교**
항목        | JPG           |PNG
---         |---            |---
항목	    |JPG	        |PNG
압축 방식	|손실 압축	    |무손실 압축
용량	    |작음	        |큼
화질	    |저장할수록 감소|원본 유지
투명 배경	|❌ 지원 안 함	|✅ 지원
사진	    |⭐ 매우 적합	|가능하지만 용량이 큼
로고/아이콘	|❌ 부적합	    |⭐ 적합
