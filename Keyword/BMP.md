# BMP
비트맵 이미지 파일 형식으로, 픽셀 하나하나의 색상 정보를 그대로 저장하는 이미지 포맷  
압축을 거의하지않기 떄문에 화질 손실이 없지만 파일 크기가 크다  

### 특징
- 확장자 : .bmp
- 개발 : Microsoft
- 압축 방식 : 대부분 무압축(일부 압축 방식 지원)
- 화질 : 원본 그대로 저장하여 손실 없음
- 파일 크기 : 매우 큼
- 투명도 : 일반적으로 지원하지 않음

### 장점
- 화질이 손상되지 않는다
- 파일 구조가 단순하여 쓰기 쉽다
- 이밎 편집 시 품질 저하가 없다

### 단점
- 용량이 매우 크다
- 인터넷 전송이나 저장 곤강 측변에서 비효율적이다
- 사진 공유용으로는 거의 사용되지 않는다

### 사용되는 곳
- Windows 프로그램 내부 이미지
- 이미지 편집 과정에서 원본 보관
- 그래픽 프로그램의 중간 작업 파일
- 픽셀 단위 이미지 처리

### BMP 파일 구조
1. 파일 헤더
    - BMP 파일인지 식별
    - 파일 크기 저장
1. 정보 헤더
    - 이미지의 가로,세로 크기
    - 색상 수
    - 압축 방식
1. 색상 테이블
    - 8비트 이하 이미지에서 색상 정보 저장
1. 픽셀 데이터
    - 실제 이미지의 픽셀 색상 정보
   
TGA와의 차이
항목     |	BMP           |	TGA
---      |---             |---
압축     |	대부분 무압축 |	무압축 또는 RLE 압축
화질     |	손실 없음     |	손실 없음
알파 채널|	거의 지원     | 안 함	지원
파일 크기|	큼            |	BMP보다 작을 수 있음(RLE 사용 시)
사용 분야|	Windows 이미지|	게임, CG, 3D 그래픽
