# AVI(Audio Video Interleave)
Microsoft가 1992년에 개발한 동영상 컨테이너 파일 형식  
'Interleave'는 오디오와 비디오 데이터를 번갈아 저장한다는 의미, 영상과 소리를 동시에 재생하기 쉽도록 설계되었다

### 특징
- 비디오와 오디오를 하나의 파일에 저장
- 다양한 비디오/오디오 코덱 지원
- Windows에서 기본적으로 지원
- 화질 손실 없이 저장 가능
- 파일 크기가 큰 편

### 컨테이너와 코덱의 차이
AVI는 컨테이너(Container) 이지 압축방식(Codec)이 아니다
```
AVI
 ├── 영상 : DivX
 ├── 영상 : Xvid
 ├── 영상 : MJPEG
 └── 오디오 : PCM, MP3
```
AVI는 데이터를 담는 '상자'이고, 코덱은 제이터를 압축, 해제하는 방식  

### 장,단점
**장점**
1. 높은 호환성
    - Windows 프로그램 대부분에서 지원한다
1. 다양한 코덱 지원
    - DivX, Xvid, MJPEG등 여러 코덱을 사용 할 수 있다
1. 높은 화질
    - 무압축 또는 낮은 압축률로 저장할 수 있어 화질이 좋다
1. 편집에 적합
    - 압축률이 낮아 영상 편집 시 품질 손실이 적다

**단점**
1. 파일 용량이 매우 큼
    - 무압축 영상은 수 GB 이상이 될 수 있다
1. 최신 기능 부족
    - 자막, 챕터, 스트리밍 등 최신기능 지원이 부족하다
1. 코덱 문제
    - 같은 AVI라도 필요한 코덱이 없으면 재생되지 않을 수 있다

### 사용되는 코덱 예시
**영상 코덱**
- CivX
- Xvid
- MJPEG
- Uncompressed
- DV

**오디오 코덱**
- PCM
- MP3
- AC3

### AVI 구조
```
AVI 파일

┌─────────────────────┐
│ Header              │
├─────────────────────┤
│ Video Data          │
├─────────────────────┤
│ Audio Data          │
├─────────────────────┤
│ Video Data          │
├─────────────────────┤
│ Audio Data          │
└─────────────────────┘
```
영상과 오디오를 번갈아 저장하여 공기화가 쉽도록 구성되어 있다

### AVI의 용도
- 영상 편집
- 화면 녹화
- 캠코더 영상 저장
- Windows 기반 동영상 저장
-  고화질 영상 보관

### AVI , MP4의 차이
AVI           | MP4
---           |---
Microsoft개발 | MPEG 개발
컨테이너 형식 | 컨테이너 형식
파일 크기가 큼| 압축 효율이 놓아 용량이 작음
편집에 유리   | 배포, 스트리밍에 유리
다양한 코덱 지원 | H.264, H.265등을 주로 사용
호환성은 좋지만 최신 기능은 부족 | 현재 가장 널리 사용되는 형식
