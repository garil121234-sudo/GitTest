# Thread
프로세스 안에서 실제로 작업을 수행하는 실행 단위  
프로세스가 하나의 실행 중인 프로그램이라면, 스레드는 그 프로그램 안에서 코드를 실행하는 작업 흐름이라고 볼 수 있다  
예를 들면 웹 브라우저는 다음과 같은 여러 스레드가 동시에 동작한다
- 화면 그리기
- 사용자 입쳑 처리
- 파일 다운로드
- 네트워크 통신
이 작업들이 각각 별도의 스레드에서 실행될 수 있다

### Process와 Thread의 관계
```
프로세스 (게임)

├── Thread 1 (렌더링)
├── Thread 2 (물리 연산)
├── Thread 3 (사운드)
├── Thread 4 (네트워크)
```
- 프로세스는 자원(메모리)을 소유한다
- 스레드는 그 자원을 공유하면서 작업을 수행한다

### Thread의 특징
1. 하나의 프로세스 안에 여러 개 존재할 수 있다
    - 이를 멀티스레드 라고한다
1. 메모리를 공유한다
    - 같은 프로세스 안의 스레드는 다음과 같은 메모리를 함께 사용한다
        - Heap
        - Data 영역
        - Code 영역
        - 하지만 각각의 스레드는 자신만의 Stack을 가진다
```
프로세스

Code
Data
Heap
 ├──────────────┐
 │              │
 │ 공유 메모리   │
 │              │
 └──────────────┘

Thread A
 Stack

Thread B
 Stack

Thread C
 Stack
```
1. 동시에 실행되는 것처럼 보인다
    - CPU가 여러 스레드를 매우 빠르게 번갈아 실행하여 동시에 실행되는 것처럼 보이게 한다
    (CPU 코어가 여러개라면 실제로 여러 스레드가 동시에 실행될 수 있다)
    
### Thread가 필요한 이유
게임에서 예를 들면
```
임 실행

↓

캐릭터 이동

↓

배경음악 재생

↓

네트워크 통신

↓

AI 계산
```
이 모든 작업을 하나의 스레드에서 처리하면 한 작업이 오래 걸릴 경우 나머지 작업이 모두 멈춘다  
멀티스레드를 사용하면
```
Thread 1 → 화면 출력

Thread 2 → 사운드 재생

Thread 3 → AI 계산

Thread 4 → 네트워크

```
각 작업을 병렬 또는 번갈아 처리하여 프로그램이 더 부드럽게 동작한다

### 장, 단점
**장점**
- 여러 작업을 동시에 처리할 수 있다
- CPU 자원을 효율적으로 사용할 수 있다
- 프로그램의 응답성이 좋아진다
- 메모리를 공유하므로 프로세스보다 생성 비용이 적다

**단점**
1. 동기화 문제  
    여러 스레드가 같은 데이터를 동시에 수정하면 예상하지 못한 결과가 발생할 수 있다  
    예를 들면
    ```
    현재 돈 = 1000원

    Thread A : +500원

    Thread B : -300원
    ```
    두 스레드가 동시에 작업하면 최종 금액이 잘못 계산될 수 있다  
    이를 해결하려면 Iock, Mutex, Semaphore 등의 동기화 기법을 사용한다
     
1. 디버깅이 어렵다  
    여러 스레드가 동시에 실행되므로 버그를 재현하기 어렵고 원인을 찾기 힘들 수 있다  

1. 교착 상태  
    스레드들이 서로 자원을 기다리며 아무도 작업을 진행하지 못하는 상태가 발생할 수 있다

### C#에서 Thread 생성
```C#
using System;
using System.Threading;

class Program
{
    static void Print()
    {
        Console.WriteLine("새로운 스레드 실행");
    }

    static void Main()
    {
        Thread thread = new Thread(Print);
        thread.Start();

        Console.WriteLine("메인 스레드 실행");
    }
}
```
Main()이 실행되는 스레드를 메인스레드  라고 하며,  
Thread.Start()를 호출하면 새로운 스레드가 생성되어 Print()메서드를 실행한다

### Process 와 Thread 비교
Process | Thread
---|---
실행중인 프로그램         | 작업을 수행하는 실행 단위
독립적인 메모리 공간 사용 | 같은 프로세스의 메모리를 공유
생성 비용이 큼            | 생성 비용이 작음
다른 프로세스와 메모리 공유가 어려움 | 같은 프로세스 내 스레드끼리 메모리 공유가능
하나 이상의 스레드를 포함 | 반드시 하나의 프로세스에 속함
