# PCB(Process Control Block)
운영체제가 각 프로세스를 관리하기 위해 사용하는 정보 저장 구조체이다  
운영체제는 프로세스가 생성될 때 PCB를 만들고, 프로세스가 종료되면 PCB를 삭제한다  

### PCB역할
CPU는 한 번에 하나의 프로세스만 실행한다(코어 1개 기준)
여러 프로세스를 동시에 실행하는 것 처럼 보이기 위해 운영체제는 매우 빠르게 프로세스를 번갈아 실행하는데,  
이때 현재 실행 중인 프로세스의 상태를 저장하고 나중에 다시 복원해야 한다  
이 역할을 하는 것이 PCB이다

### PCB에 저장되는 정보
- 프로세스 ID(PID) : 프로세스를 구분하는 고유 번호
- 프로세스 상태(State) : Ready, Running, Waiting, Terminated 등
- 프로그램 카운터(PC) : 다음에 실행할 명령어의 주소
- CPU 레지스터 값 : 현재 작업중인 레지스터 값
- 메모리 관리 정보 : 코드, 데이터, 스택, 힙 등의 메모리 정보
- CPU 스케줄링 정보 : 우선순위, 스케줄링 큐 정보
- 입출력(I/O) 정보 : 사용 중인 파일, 장치 정보
- 계정 정보(Accunting) : CPU 사용 시간 등

### 예시
PCB는 게임의 저장 파일과 비슷하다
```
프로세스 실행 (게임 플레이)
      ↓
PCB에 현재 상태 저장 (게임 저장)
      ↓
다른 프로세스 실행
      ↓
PCB에서 상태 복원
      ↓
이전 위치부터 계속 실행 (나중에 불러오기)
```

### Process 와 PCB의 관계
Process
- Code
- Data
- Heap
- Stack
- PCB(운영체제가 관리하는 정보)
    - PID
    - State
    - Register
    - PC
    - Priority
    - Memory Info

※프로세스 자체(코드, 데이터, 힙, 스택)와 PCB는 구분된다.
PCB는 운영체제가 프로세스를 관리하기 위해 별도로 유지하는 관리 정보이다
